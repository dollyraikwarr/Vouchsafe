The problem

Getting paid for real work still runs on trust

If you're the developer

You deliver first and invoice after. If the client stalls or disappears, you have no leverage and no record that says otherwise.

If you're the client

You pay upfront and hope, or you route everything through a platform that takes a cut and decides on its own terms when money moves.

Vouchsafe replaces both with a state machine: an on-chain escrow that only two actions can move — the developer submitting proof, and the client approving it.

How it works

Four steps. One state machine. No exceptions.

01

Create

The client defines the engagement — developer address, payment amount, token, and deadline.

02

Fund

Payment locks into the contract's on-chain escrow. Nobody can spend it until the state machine moves.

03

Submit

The developer records deliverable proof on-chain: a URL, a PR link, a commit hash, and a note.

04

Approve

The client approves. Payment releases to the developer automatically, in the same transaction.

On-chain record

Every engagement is a permanent, queryable record

get_engagement() returns exactly what happened — client, developer, amount, deliverable, and whether payment actually released.
